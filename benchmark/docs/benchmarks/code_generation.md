# Code Generation Benchmark
The code generation benchmark evaluates the ability of LLMs to generate functionally correct and executable code.

The evaluation focuses on:
- Code generation accuracy (functional correctness)
- End-to-end latency during inference

## Benchmark Methods
### Dataset: HumanEval, MBPP (via EvalPlus)
We evaluate code generation performance on the HumanEval and MBPP datasets using the EvalPlus framework.

- EvalPlus Paper: [Is Your Code Generated by ChatGPT Really Correct? Rigorous Evaluation of Large Language Models for Code Generation](https://openreview.net/forum?id=1qvx610Cu7)
- Evaluation Code: [EvalPlus (GitHub)](https://github.com/evalplus/evalplus)

HumanEval evaluates a model's ability to generate code that passes unit tests, while MBPP evaluates a model's ability to synthesize entry-level Python programs from natural language instructions.

- HumanEval Paper: [Evaluating Large Language Models Trained on Code](https://arxiv.org/abs/2107.03374)
- MBPP Paper: [Program Synthesis with Large Language Models](https://arxiv.org/abs/2108.07732)

### Running the Benchmark
The required benchmark framework is automatically installed when the CLI is executed with the `humaneval` or `mbpp` task specified in the configuration file.
Example configuration (`my_benchmark.yaml`):
```yaml
benchmark:
  # The benchmark task to run
  task: humaneval # or mbpp

```

### Environment setups
The installation script is located at: [install_evalplus.sh](https://github.com/furiosa-ai/furiosa-apps/blob/main/furiosa-bench/packages/furiosa-bench-launcher/furiosa_bench_launcher/metadata/benchmark/scripts/install_evalplus.sh).

It installs the required evaluation dependencies:
```bash
pip install uv
uv venv .evalplus_venv
uv pip install --python .evalplus_venv/bin/python "evalplus[vllm]" --upgrade

```

### Run tasks
Benchmark task definitions are located at: [task.yaml](https://github.com/furiosa-ai/furiosa-apps/blob/main/furiosa-bench/packages/furiosa-bench-launcher/furiosa_bench_launcher/metadata/benchmark/tasks.yaml)
Example task configuration:

```yaml
- humaneval:
    description: Functional correctness evaluation for code generation models
    framework: evalplus
    defaults:
      command: >
        ../.evalplus_venv/bin/python -m evalplus.evaluate
        --model {{deployment.model_id}}
        --dataset {{benchmark.task}}
        --base_url http://{{deployment.host}}:{{deployment.port}}/v1
        --backend openai --greedy

- mbpp:
    description: Mostly Basic Python Problems (MBPP) evaluation.
    framework: evalplus
    defaults:
      command: >
        ../.evalplus_venv/bin/python -m evalplus.evaluate
        --model {{deployment.model_id}}
        --dataset {{benchmark.task}}
        --base_url http://{{deployment.host}}:{{deployment.port}}/v1
        --backend openai --greedy
```
